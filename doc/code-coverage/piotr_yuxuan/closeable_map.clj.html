089  
090  When `(.close system)` is executed, it will:
091  
092    - Recursively close all instances of `java.io.Closeable` and
093      `java.lang.AutoCloseable`;
094    - Recursively call all stop zero-argument functions tagged with
095      `^::closeable-map/fn`;
096    - Skip all nested `Closeable` under a `^::closeable-map/ignore`;
097    - Silently swallow any exception with `^::closeable-map/swallow`;
098    - Exceptions to optional `::closeable-map/ex-handler` in key or
099      metadata;
100    - If keys (or metadata) `::closeable-map/before-close` or
101      `::closeable-map/after-close` are present, they will be assumed as
102      a function which takes one argument (the map itself) and used run
103      additional closing logic:
104  
105      ``` clojure
106      (closeable-map
107        {;; This function will be executed before the auto close.
108         ::closeable-map/before-close (fn [this-map] (flush!))
109  
110         ;; Kafka producers/consumers are java.io.Closeable
111         :producer (kafka-producer config)
112         :consumer (kafka-consumer config)
113  
114         ;; This function will be executed after the auto close.
115         ::closeable-map/after-close (fn [this-map] (garbage/collect!))})
116      ```
117

119  some similar method. For example instances of
120  `java.util.concurrent.ExecutorService` can't be closed but they can be

147  ## All or nothing
148  
149  ### No half-broken closeable map
150  
151  You may also avoid partially open state when an exception is thrown
152  when creating a `CloseableMap`. This is where `closeable-map*` comes
153  handy. It outcome in one of the following:
154  
155  - Either everything went right, and all inner forms wrapped by
156    `closeable` correctly return a value; you get an open instance of `CloseableMap`.
157  
158  - Either some inner form wrapped by `closeable` didn't return a
159    closeable object but threw an exception instead. Then all
160    `closeable` forms are closed, and finally the exception is
161    bubbled up.
162  
163  ``` clojure
164  (closeable-map*
165    {:server (closeable* (http/start-server (api config)))
166     :kafka {:consumer (closeable* (kafka-consumer config))
167             :producer (closeable* (kafka-producer config))
168             :schema.registry.url \"https://localhost\"}})
169  ```
170  
171  ### No half-broken state in general code
172  
173  In some circumstances you may need to handle exception on the creation
174  of a closeable map. If an exception happens during the creation of the
175  map, values already evaluated will be closed. No closeable objects
176  will be left open with no references to them.
177  
178  For instance, this form would throw an exception:
179  
180  ``` clojure
181  (closeable-map/closeable-map {:server (http/start-server (api config))
182                                :kafka {:consumer (kafka-consumer config)
183                                        :producer (throw (ex-info \"Exception\" {}))}})
184  ;; => (ex-info \"Exception\" {})
185  ```
186  
187  `with-closeable*` prevents that kind of broken, partially open states for its bindings:
188  
189  ``` clojure
190  (with-closeable* [server (http/start-server (api config))
191                    consumer (kafka-consumer config)
192                    producer (throw (ex-info \"Exception\" {}))]
193    ;; Your code goes here.
194  )
195  ;; Close consumer,
196  ;; close server,
197  ;; finally throw `(ex-info \"Exception\" {})`.
198  ```
199  
200  You now have the guarantee that your code will only be executed if
201  all these closeable are open. In the latter example an exception is
202  thrown when `producer` is evaluated, so `consumer` is closed, then
203  `server` is closed, and finally the exception is bubbled up. Your
204  code is not evaluated. In the next example the body is evaluated,
205  but throws an exception: all bindings are closed.
206  
207  ``` clojure
208  (with-closeable* [server (http/start-server (api config))
209                    consumer (kafka-consumer config)
210                    producer (kafka-producer config)]
211    ;; Your code goes here.
212    (throw (ex-info \"Exception\" {})))
213  ;; Close producer,
214  ;; close consumer,
215  ;; close server,
216  ;; finally throw `(ex-info \"Exception\" {})`.
217  ```
218  
219  When no exception is thrown, leave bindings open and return like a
220  normal `let` form. If you prefer to close bindings, use `with-open` as
221  usual.
222    
223  ``` clojure
224  (with-closeable* [server (http/start-server (api config))
225                    consumer (kafka-consumer config)
226                    producer (kafka-producer config)]
227    ;; Your code goes here.
228    )
229  ;; All closeable in bindings stay open.
230  ;; => result
231  ```"

312  (def visitor
313    "Take a form `x` as one argument and traverse it while trying to
314    [[piotr-yuxuan.closeable-map/close!]] inner items.
315  
316    Map keys `::before-close` and `::after-close` will be invoked before
317    and after other keys of the map. Will ignore items marked with
318    `^::ignore`. Exceptions when closing some item will be passed to an

320    higher and will stay silently swallowed. Functions tagged with
321    `^::fn` will be considered closeable and will thus be called on
322    `.close`.

325    requirement is they be invokable. Map keys can also be tagged. Maps
326    values `::before-close` and `::after-close` are not expected to be

471  (defmacro with-closeable*
472    "Take two arguments, a `bindings` vector and a `body`. Like a `let`,
473    support destructuring. Avoids partially open state when an exception
474    is thrown on evaluating closeable forms. Evaluate bindings
475    sequentially then return `body` and leave bindings open. When an
476    exception is thrown in a later binding or in the `body`, close
477    bindings already open in reverse order and finally bubble up the
478    exception. Do nothing for non-closeable bindings.
479  
480    Use it if you need exception handling on the creation of a closeable
481    map, so no closeable objects are left open but with no references
482    because of an exception.
483  
484    For instance, this form would throw an exception and leave the
485    server open and the port locked:
486  
487    ``` clojure
488    (closeable-map {:server (http/start-server (api config))
489                    :kafka {:consumer (kafka-consumer config)
490                            :producer (throw (ex-info \"Exception\" {}))}})
491    ;; `consumer` and `server` stay open but with no reference. Kafka
492    ;; messages are consumed and the port is locked.
493    ;; => (ex-info \"Exception\" {})
494    ```
495  
496    `with-closeable*` prevents that kind of broken, partially open
497    states for its bindings:
498  
499    ``` clojure
500    (with-closeable* [server (http/start-server (api config))
501                      consumer (kafka-consumer config)
502                      producer (throw (ex-info \"Exception\" {}))]
503      ;; Your code goes here.
504    )
505    ;; Close consumer,
506    ;; close server,
507    ;; finally throw `(ex-info \"Exception\" {})`.
508    ```
509  
510    You now have the guarantee that your code will only be executed if
511    all these closeable are open. In the latter example an exception is
512    thrown when `producer` is evaluated, so `consumer` is closed, then
513    `server` is closed, and finally the exception is bubbled up. Your
514    code is not evaluated. In the next example the body is evaluated,
515    but throws an exception: all bindings are closed.
516  
517    ``` clojure
518    (with-closeable* [server (http/start-server (api config))
519                      consumer (kafka-consumer config)
520                      producer (kafka-producer config)]
521      ;; Your code goes here.
522      (throw (ex-info \"Exception\" {})))
523    ;; Close producer,
524    ;; close consumer,
525    ;; close server,
526    ;; finally throw `(ex-info \"Exception\" {})`.
527    ```
528  
529    When no exception is thrown, leave bindings open and return like a
530    normal `let` form.
531    
532    ``` clojure
533    (with-closeable* [server (http/start-server (api config))
534                      consumer (kafka-consumer config)
535                      producer (kafka-producer config)]
536      ;; Your code goes here.
537      )
538    ;; All closeable in bindings stay open.
539    ;; => result
540    ```"
541    [bindings & body]
542    (assert (even? (count bindings)) "Expecting an even number of forms in `bindings`.")
543    (if (zero? (count bindings))
544      `(binding [*closeables* (or *closeables* (atom ()))]
545         ~@body)
546      (let [v (gensym "value")]
547        `(binding [*closeables* (or *closeables* (atom ()))]
548           (try
549             (let [~(nth bindings 0) ~(nth bindings 1) ; unsplice?
550                   ~v ~(nth bindings 0)]
551               (swap! *closeables* conj ~v)
552               (with-closeable* ~(subvec bindings 2) ~@body))
553             (catch Throwable th#
554               (doseq [c# @*closeables*]
555                 (try (close! c#)
556                      (catch Throwable _#)
557                      (finally (swap! *closeables* rest))))
558               (throw th#)))))))

560  (defmacro ^CloseableMap closeable-map*
561    "Avoid partially open state when an exception is thrown on evaluating
562    inner closeable forms wrapped by `closeable`.
563  
564    ``` clojure
565    (closeable-map*
566      {:server (closeable* (http/start-server (api config)))
567       :kafka {:consumer (closeable* (kafka-consumer config))
568               :producer (closeable* (kafka-producer config))
569               :schema.registry.url \"https://localhost\"}})
570    ```
571  
572    The outcome of this macro `closeable-map*` is one of the following:
573  
574    - Either everything went right, and all inner forms wrapped by
575      `closeable` correctly return a value, then this macro returns an
576      open instance of `CloseableMap`.
577  
578    - Either some inner form wrapped by `closeable` didn't return a
579      closeable object but threw an exception instead. Then all
580      `closeable` forms are closed, and finally the exception is
581      bubbled up.
582  
583    Known (minor) issue: type hint is not acknowledged, you have to tag
584    it yourself with `^CloseableMap` (most precise),
585    `^java.io.Closeable`, or `^java.lang.AutoCloseable` (most generic)."
586    [m]
587    `(binding [*closeables* (or *closeables* (atom ()))]
588       (try (vary-meta (closeable-map ~m) assoc :tag `CloseableMap)
589            (catch Throwable th#
590              (doseq [c# @*closeables*]
591                (try (close! c#)
592                     (catch Throwable _#)
593                     (finally (swap! *closeables* rest))))
594              (throw th#)))))
